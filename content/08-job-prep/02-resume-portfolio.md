## Resume Best Practices

### Format and Structure

| Element | Recommendation |
|---------|---------------|
| **Length** | 1-2 pages maximum |
| **Format** | Clean, scannable, ATS-friendly |
| **Template** | Simple, avoid tables/graphics |
| **File Type** | PDF or .docx |

### Essential Sections

1. **Contact Information**
   - Name, phone, email
   - LinkedIn profile URL
   - **GitHub profile URL** (critical for technical roles)
   - Portfolio website (if applicable)

2. **Professional Summary**
   - 3-4 lines maximum
   - Focus on where you're going, not where you've been
   - Include key certifications
   - Mention specialization interest

3. **Technical Skills**
   - Organize by category: Tools, Frameworks, Languages, Platforms
   - Include certifications with acronyms AND full names
   - Match keywords from job descriptions

4. **Work Experience**
   - Quantify achievements ("Reduced incident response time by 40%")
   - Focus on security-related responsibilities
   - Use action verbs

5. **Education and Certifications**
   - Relevant degrees
   - Industry certifications
   - Online courses (Google, Coursera, etc.)

6. **Projects**
   - Link to GitHub or portfolio
   - Brief description of what you built

### 2026 Resume Trends

- **AI Integration:** 10% of cybersecurity job listings specifically reference AI skills
- **Skills-Based Hiring:** Experience over degrees alone
- **GitHub Links:** Essential for demonstrating practical work
- **ATS Optimization:** Keywords determine if humans see your resume

## ATS Optimization

### What is ATS?

Applicant Tracking Systems filter resumes before humans see them. **75% of resumes are rejected by ATS** before reaching a recruiter.

### ATS-Friendly Tips

| Do This | Avoid This |
|---------|-----------|
| Standard section headings | Creative headings |
| Simple formatting | Tables, graphics, headers/footers |
| PDF or .docx | Images, .txt, .pages |
| Full certification names + acronyms | Just acronyms |
| Keyword matching from job descriptions | Generic copy-paste |

### 2026 Keywords to Include

| Category | Keywords |
|----------|----------|
| **Frameworks** | NIST CSF 2.0, OWASP Top 10, MITRE ATT&CK |
| **Tools** | Splunk, Sentinel, Wireshark, Nmap, Metasploit |
| **Skills** | Risk Management, Threat Intelligence, Incident Response, Vulnerability Assessment |
| **Concepts** | Zero Trust Architecture, Cloud Security, SIEM, EDR/XDR |
| **Languages** | Python, Bash, PowerShell, SQL |
| **Cloud** | AWS, Azure, GCP, Kubernetes |
| **Compliance** | GDPR, SOC 2, ISO 27001, PCI DSS |

### Industry Frameworks to Reference

- **NIST CSF 2.0** — https://www.nist.gov/cyberframework
- **OWASP Top 10 2025** — https://owasp.org/www-project-top-ten/

## Portfolio Projects

### What Employers Want to See

Portfolio projects demonstrate you can actually do the work, not just pass exams.

### High-Value Project Categories

#### A. Home Lab Projects

| Project | Tools | Demonstrates |
|---------|-------|--------------|
| SIEM Setup | Splunk Free, ELK Stack, Wazuh | Log analysis, threat detection |
| Network Security Lab | pfSense, VLANs, Firewalls | Network segmentation, ACLs |
| Threat Detection System | Snort, Suricata, Zeek | IDS/IPS configuration |
| Vulnerability Scanner | OpenVAS, Nessus | Assessment capabilities |

#### B. CTF and Practice Platforms

- **Hack The Box** — https://www.hackthebox.com
- **TryHackMe** — https://tryhackme.com
- **PortSwigger Web Security Academy** — https://portswigger.net/web-security (FREE)

**Documentation:** Write up solutions and post to GitHub. Shows technical writing and problem-solving.

#### C. GitHub Project Examples

Based on successful cybersecurity portfolios:

- **Python automation scripts** for security tasks
- **Log analyzer tools** — Parse and visualize security logs
- **Network architecture designs** — VLAN segmentation, firewall rules
- **API security research labs** — Educational exploits for learning
- **Cloud security projects** — AWS/Azure security configurations

### Project Documentation Template

Each project should include:

```markdown
# Project Name

## Objective
What problem does this solve?

## Technologies Used
- Python 3.12
- Splunk Free
- VirtualBox

## Implementation
Step-by-step explanation

## Results
Screenshots, metrics, outcomes

## Lessons Learned
What you discovered
```

## LinkedIn Profile Optimization

### Profile Essentials

1. **Headline**
   - Include "Cybersecurity" + specialization + key credential
   - Example: "SOC Analyst | Security+ | Threat Detection & Incident Response"

2. **About Section**
   - Professional summary with keywords
   - 3-4 paragraphs
   - Include certifications and specializations
   - Call to action ("Open to opportunities")

3. **Featured Section**
   - Link to portfolio
   - Link to GitHub
   - Link to certifications

4. **Skills**
   - Top 3: Most relevant technical skills
   - Get endorsements from peers

5. **Recommendations**
   - From peers, managers, course instructors
   - Even 2-3 help significantly

### Content Strategy

- Share weekly insights on security news
- Document learning journey (certifications, CTFs)
- Engage with cybersecurity community posts
- Follow target companies and security leaders
